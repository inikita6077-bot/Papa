Tratarea Excepțiilor
în C#

Ghid complet pentru gestionarea erorilor și crearea de aplicații robuste

1. Ce Sunt Excepțiile?

O excepție este o eroare care apare în timpul execuției programului și întrerupe fluxul normal al acestuia. În loc să permiteți programului să se blocheze brutal, C# oferă mecanisme de tratare gracioasă a acestor erori.

De ce sunt importante?

✓ Asigură stabilitatea aplicației
✓ Permite gestionarea controlată a erorilor
✓ Îmbunătățește experiența utilizatorului
✓ Facilitează debugging-ul codului

2. Tipuri de Excepții în C#

C# oferă multe clase de excepții predefinite pentru diferite situații:

DivideByZeroException

Se ridică când încercați să împărțiți un număr la zero.

FormatException

Apare la conversii de tip greșite, de exemplu text în loc de număr.

NullReferenceException

Accesarea unui obiect care este null (nul).

IndexOutOfRangeException

Index invalid atunci când accesați un element dintr-un array.

3. Structura Try-Catch-Finally

Aceasta este baza tratării excepțiilor în C#. Trei blocuri fundamentale:

1 try
2 catch
3 finally

try: Conține codul care poate genera eroare

catch: Tratează excepția dacă este ridicată

finally: Se execută întotdeauna, indiferent de rezultat

try { // Cod care poate genera eroare int a = 10; int b = 0; int c = a / b; // DivideByZeroException! } catch (DivideByZeroException) { Console.WriteLine("Eroare: Împărțire la 0!"); } finally { Console.WriteLine("Operație finalizată"); }

4. Exemplu Practic: Validare Input Utilizator

Unul dintre cazurile cele mai frecvente este prevenirea blocării programului când utilizatorul introduce date invalide:

try { Console.Write("Introdu un număr: "); int x = int.Parse(Console.ReadLine()); Console.WriteLine("Numărul: " + x); } catch (FormatException) { Console.WriteLine("❌ Ai introdus text, nu număr!"); } catch (Exception ex) { Console.WriteLine("❌ Eroare: " + ex.Message); }

Observație Importantă

Puteți avea mai mulți catch! Fiecare tratează o eroare diferită. Cel mai specific se execută primul. Ordinea contează!

5. Excepții Personalizate (Custom Exceptions)

Puteți crea propriile excepții pentru reguli de business specifice. De exemplu, un utilizator nu poate avea vârstă negativă:

public class InvalidAgeException : Exception { public InvalidAgeException(string msg) : base(msg) { } } static void SetAge(int age) { if (age < 0) { throw new InvalidAgeException( "Vârsta nu poate fi negativă!"); } Console.WriteLine("Vârsta: " + age); }

Avantaje: Cod mai curat fără mii de if-uri, separare logică clară a validării, mesaje specifice pentru domeniu de aplicație.

6. Avantajele Tratării Excepțiilor

Stabilitate

Aplicația nu se blochează complet. Programul poate continua să funcționeze și să se recupereze din erori.

UX Bună

Utilizatorii primesc mesaje clare și înțelese despre ce s-a întâmplat, nu erori criptice.

Debugging Ușor

Proprietatea ex.Message ajută la identificarea rapidă și rezolvarea problemelor.

Control Fluxu

Decideți exact cum programul va continua după o eroare.

Protecție Resurse

Blocul finally asigură eliberarea fișierelor, conexiunilor de bază de date și memoriei.

Cod Curat

Separare clară între logica normală și tratarea erorilor.

7. Legatura cu Programarea Orientată pe Obiecte

Excepțiile nu sunt doar mecanisme de eroare—sunt obiecte în C# și demonstrează puterea OOP:

Principiile OOP în Excepții

Încapsulare: Protejați datele și validați prin excepții.
Moștenire: Excepțiile personalizate moștenesc din clasa Exception.
Polimorfism: Un catch cu base Exception poate prinde toate derivatele.

Aceasta demonstrează că un programator serios creează cod robust prin tratarea excepțiilor. Codul fără tratare = codul amator. Codul cu tratare = profesional și stabil.

Concluzie

Tratarea excepțiilor în C# nu este opțională—este marca unui programator serios. Prin înțelegerea și implementarea corectă a try-catch-finally, a excepțiilor personalizate și a best practices, creați aplicații care sunt stabile, ușor de întreținut și care oferă o experiență excelentă utilizatorului.
